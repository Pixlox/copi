# Releases

Copi now uses Tauri's signed updater against GitHub Releases for macOS and Windows.

The app checks this endpoint:

`https://github.com/Pixlox/copi/releases/latest/download/latest.json`

That `latest.json` file is generated by the release workflow and points the app at the newest signed bundle for the current platform.

## One-time Setup

1. Generate an updater signing key.

```bash
npm run tauri signer generate -- -w ~/.tauri/copi.key
```

2. Keep the private key outside the repo.

Recommended location:

`~/.tauri/copi.key`

Do not place the private key anywhere inside this repository.

3. Copy the generated public key into `src-tauri/tauri.conf.json` under `plugins.updater.pubkey`.

4. Add these GitHub repository secrets:

- `TAURI_SIGNING_PRIVATE_KEY`: the contents of your private key file.
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`: the password used when you created the key, if any.

5. Keep the private key backed up somewhere safe. If you lose it, existing installs will no longer trust future updates signed by a replacement key.

The private key password should live only in your local shell environment when signing locally, or in GitHub Actions secrets when signing in CI. Do not commit it to `.env`, `.toml`, `.json`, or workflow files.

## Preparing a Release

1. Bump every app version in one shot:

```bash
npm run release:version -- 0.1.1
```

2. Run the release sanity check:

```bash
npm run release:check
```

3. Commit the version change.

```bash
git add package.json package-lock.json src-tauri/Cargo.toml src-tauri/tauri.conf.json
git commit -m "Release v0.1.1"
```

4. Tag the release and push it:

```bash
git tag v0.1.1
git push origin main --follow-tags
```

## What GitHub Does

Pushing a `v*` tag triggers `.github/workflows/release.yml`.

That workflow:

- verifies the versions match the tag
- verifies the updater public key is not still a placeholder
- builds macOS Apple Silicon and Windows bundles
- signs updater artifacts with your Tauri private key
- uploads bundles, signatures, and `latest.json` to a published GitHub Release

## Notes

- Intel macOS is not part of the release matrix. The ONNX Runtime crate used for semantic search does not ship a prebuilt `x86_64-apple-darwin` runtime, so Copi now releases Apple Silicon macOS builds only.
- Tauri updater signing is separate from Apple notarization and Windows code signing. The updater will work without those, but OS trust prompts may still be rougher for end users.
- If the app reports an invalid signature, the public key in `src-tauri/tauri.conf.json` does not match the private key in GitHub secrets.
